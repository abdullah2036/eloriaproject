# 🪶 Eloria Story — دليل الاستخدام

موقعك كله **4 ملفات فقط**، بلا أدوات برمجية ولا تثبيت أي شيء:

| الملف | وظيفته |
|---|---|
| `index.html` | هيكل الصفحات |
| `style.css` | التصميم والألوان |
| `app.js` | الأزرار ووضع المؤلفة (**كلمة السر في أول سطوره** 🗝) |
| `data.json` | المحتوى المنشور (الروايات، الفصول، الشخصيات، الاقتباسات) |

---

## 1) رفع الموقع على GitHub Pages (مرة واحدة)

1. أنشئي حسابًا على github.com ثم اضغطي **New repository**، وسمّيه مثلًا `eloria`.
2. داخل المستودع اضغطي **Add file → Upload files** واسحبي الملفات الأربعة (`index.html`, `style.css`, `app.js`, `data.json`) ثم **Commit changes**.
3. من **Settings → Pages** اختاري: Branch = `main`، المجلد = `/ (root)` ثم **Save**.
4. بعد دقيقة يصبح موقعك على: `https://اسمك.github.io/eloria/`

لا حاجة لأي terminal أو أوامر — كل شيء بالسحب والإفلات من المتصفح.

---

## 2) دخول وضع المؤلفة ✒

- اضغطي أيقونة المفتاح 🗝 أعلى الصفحة وأدخلي كلمة السر.
- كلمة السر الحالية: `eloria` — **غيّريها** بفتح `app.js` — ستجدينها في أول الملف داخل إطار واضح:
  `const AUTHOR_PASS = "eloria";`
- عند التفعيل تظهر لك أزرار: رواية جديدة، فصل جديد، شخصية، اقتباس، تعديل، حذف، ومربعات سحب وإفلات لملفات `.txt` و`.md` (ملف واحد داخل نافذة الفصل، أو عدة ملفات دفعة واحدة في صفحة الرواية).

ملاحظة صريحة: هذه حماية بسيطة لإخفاء أزرار التحرير عن الزوار، وليست نظام حسابات حقيقيًا — وهي مناسبة تمامًا لموقع تنشرين فيه محتوى عامًا، لأن النشر الفعلي لا يتم إلا بيدك (الخطوة التالية).

---

## 3) النشر للزوار — بضغطة زر 🚀

كل تعديلاتك تُحفظ فورًا **في متصفحك** كمسودّات آمنة. وحين تجهزين، اضغطي **🚀 نشر للزوار** من شريط أدوات النشر — وخلال دقيقة يرى الجميع المحتوى الجديد. تمامًا كحفظ فصل.

### إعداد لمرة واحدة فقط (٣ دقائق)
زر النشر يحتاج «مفتاح نشر» يسمح لموقعك بالحفظ في مستودعك على GitHub:

1. افتحي **github.com/settings/personal-access-tokens** واضغطي **Generate new token**.
2. تحت **Repository access** اختاري **Only select repositories** ثم مستودع موقعك فقط.
3. تحت **Permissions → Contents** اختاري **Read and write**، ثم أنشئي المفتاح وانسخيه.
4. في موقعك: **⚙ إعداد النشر** → ألصقي رابط مستودعك والمفتاح → **حفظ الإعداد**.

🔒 المفتاح يُحفظ في متصفحك أنت فقط ولا يُنشر مع الموقع أبدًا. إن تسرّب يومًا، يكفي حذفه من صفحة GitHub نفسها وإنشاء غيره.

### احتياط دائم
- **⬇ نسخة احتياطية**: تنزّل `data.json` لجهازك — احتفظي بنسخة بين حين وآخر.
- **⬆ استيراد**: تُرجع أي نسخة محفوظة إلى أي متصفح.
- وإن تعطّل زر النشر يومًا (انقطاع إنترنت مثلًا)، الطريقة اليدوية القديمة تعمل دائمًا: نزّلي النسخة الاحتياطية وارفعيها إلى المستودع من صفحة GitHub بالسحب والإفلات.

## أسئلة سريعة

- **أريد المعاينة قبل الرفع؟** افتحي `index.html` بالنقر المزدوج — يعمل مباشرة.
- **أغلفة الروايات؟** ترفعينها من نافذة «رواية جديدة/تعديل»، وتُضغط تلقائيًا لتبقى خفيفة وتُحفظ داخل `data.json` نفسه.
- **الموقع بطيء أو الملف كبر كثيرًا؟** استخدمي أغلفة أصغر؛ الضغط التلقائي يتكفل بالباقي غالبًا.
- **نسيت كلمة السر؟** افتحي `app.js` واقرئي قيمة `AUTHOR_PASS`.
